# 회의록 보조 프로그램 (Meeting Assistant)

브라우저에서 실행되는 회의 보조 도구입니다.  
실시간 음성 인식 + AI 요약(Gemini) + 리치 텍스트 메모 + 녹음 파일 저장을 한 번에 제공합니다.

### 0. 실행 방법
1. pj1.html 파일을 폴더에 저장
2. 터미널에서 파일이 들어있는 디렉터리로 들어간 후, python -m http.server 8000 입력.(서버 실행)
3. 크롬 주소창에  http://localhost:8000/pj1.html 입력

---

## 1. 개발 배경 및 필요성

### 1-1. 회의에서 자주 겪는 문제들

- 회의에 집중하느라 **메모를 놓치거나**, 메모하느라 **발언을 놓치는** 상황
- 회의가 끝난 뒤  
  - “**결국 뭐 하기로 한 거지?**”  
  - “**누가 뭐 한다고 했지?**”  
  를 다시 물어보는 상황
- 한 번만 얘기했는데, 나중에  
  - “그거 말한 적 없다” / “그렇게 정리된 줄 몰랐다”  
  같은 **커뮤니케이션 오류** 발생
- 회의록을 담당한 사람이 회의 이후에  
  - 녹음 다시 듣고  
  - 타이핑하고  
  - 정리까지 해야 하는 **추가 노동**

### 1-2. 이 프로그램이 해결하려는 것

이 프로그램은 회의 중에 다음을 자동으로 처리해 줍니다.

- **실시간으로 대화 내용을 텍스트로 기록**  
- 주제가 하나 끝날 때마다 버튼 한 번으로
  - **Gemini가 핵심 내용·결정사항·액션아이템 요약**
- 동시에 오른쪽에서 **직접 메모/강조/정리** 가능
- 회의가 끝나면
  - **요약본 → PDF**
  - **메모 → TXT**
  - **전체 녹음 → WEBM**
  로 한 번에 저장

즉, “**회의에 집중하면서도, 회의 기록과 정리까지 자동으로 챙기는 것**”이 목적입니다.

## 2. 구현 기능

### 2-1. 실시간 음성 인식 및 회의 기록

- 브라우저(Web Speech API)를 활용한 **실시간 음성 인식**
- 사용 기술:
  - `SpeechRecognition / webkitSpeechRecognition`
  - `continuous: true`, `interimResults: true` 설정으로 **가능한 한 연속 인식**
- 화면 구성:
  - 왼쪽 상단 “실시간 대화 내용” 영역에
    - 시간 + 발언 내용이 순서대로 누적
    - 인식 중인 문장은 *[인식 중] …* 형태로 표시
- 안정성을 위해:
  - 일정 시간 동안 결과가 없으면 자동으로 인식을 재시도하는 **watchdog 로직** 포함
  - 세션 종료 시 `onend`에서 자동 재시작 시도

> ⚠️ 음성 인식은 브라우저(Web Speech API) 한계에 따라  
> 완전한 “무제한 스트리밍”은 아니며, 환경에 따라 일부 끊김이 있을 수 있습니다.  
> (크롬 + localhost/https 에서 가장 안정적으로 동작)

---

### 2-2. Gemini 기반 중간 요약 기능

- **주제 하나가 마무리될 때** “중간 정리” 버튼 클릭
- 직전 대화 내용(또는 전체 기록)을 모아 Gemini API에 전달
- 모델:
  - `gemini-2.5-flash` 사용
- 입력:
  - 회의 텍스트를 기반으로
  - “핵심 논의, 결정 사항, 액션 아이템을 불릿 목록으로 정리해 달라”는 프롬프트
- 출력:
  - 왼쪽 하단 “주제별 중간 정리” 영역에 카드 형태로 누적
  - 각 카드:
    - `주제 N 정리` + 생성 시각 + 요약 내용

#### Gemini 키 유무에 따른 동작

- **API 키 입력 O**  
  → Gemini가 회의 내용을 이해해서 구조적인 요약을 생성
- **API 키 입력 X 또는 오류 발생**  
  → 자체 **간단 요약(naive summarize)** 로 대체해서  
    항상 어떤 형태로든 요약 텍스트를 보여줌

---

### 2-3. 리치 텍스트 메모장

- 화면 오른쪽은 **회의 메모 전용 에디터**
- 지원 기능:
  - 글꼴 선택 (기본/나눔고딕/돋움 등)
  - 글자 크기: **1pt ~ 30pt** 직접 입력
  - 글자 색상 선택 (color picker)
  - 형광펜 색상 선택 (hiliteColor)
  - 서식:
    - 굵게(B), 기울임(I), 밑줄(U), 취소선
    - 번호 없는 목록(•), 번호 있는 목록(1.)
  - 메모 전체 지우기 버튼
- 형광펜 취소 기능:
  - 형광펜이 칠해진 영역을 마우스로 **드래그 선택**
  - “형광펜 해제” 버튼 클릭
  - 선택 범위 안의 요소들 중 **background / background-color 스타일만** 제거하여
    - 다른 스타일(굵게, 색, 크기 등)은 유지

---

### 2-4. 저장 기능 (요약 PDF + 메모 TXT + 녹음 WEBM)

- 상단 “저장” 버튼 클릭 시, 아래 3가지 파일을 자동 생성:

1. **회의 요약본 (PDF)**
   - `html2pdf.js`, `html2canvas`, `jsPDF` 사용
   - 왼쪽 하단 “주제별 중간 정리” 영역만 캡처해서
   - `회의_요약본.pdf` 로 저장

2. **회의 메모 (TXT)**
   - 메모 에디터 내부의 순수 텍스트를 추출
   - `회의_메모.txt` 파일로 저장

3. **회의 녹음 (WEBM)**
   - `MediaRecorder`로 녹음된 오디오를 Blob으로 변환
   - `회의_녹음.webm` 파일로 저장

> 요약본이 하나도 없으면  
> → “아직 요약이 없습니다. 메모와 녹음만 저장합니다.” 알림 후  
> 메모 TXT와 녹음 WEBM만 저장합니다.

---

### 2-5. 다크 모드 / 라이트 모드

- `data-theme="light" | "dark"` 토글 방식
- 상단 우측 버튼(🌙 / ☀️)으로 모드 전환 가능
- 색상 팔레트는 CSS 변수(`--bg-color`, `--text-color` 등)로 분리

## 3. 동작 환경 및 기술 스택

### 3-1. 동작 환경

- **브라우저**: 데스크톱 크롬 권장
- **주소**:
  - `file://` 로 여는 것보다  
    `http://localhost:8000` 같은 **로컬 서버 환경** 권장
- **마이크 권한**:
  - 최초 접속 시 “마이크 사용 허용” 필요
  - 주소창 왼쪽 자물쇠 → 마이크 → “허용” 권장

### 3-2. 사용 기술

- 프론트엔드:
  - 순수 HTML, CSS, JavaScript
- 음성 관련:
  - Web Speech API (`SpeechRecognition / webkitSpeechRecognition`)
  - MediaRecorder API (`getUserMedia` + `MediaRecorder`)
- PDF 변환:
  - `html2pdf.js`
  - `html2canvas`
  - `jsPDF`
- AI 요약:
  - Google Gemini API (`gemini-2.5-flash`, `v1beta generateContent`)

## 4. 기대 효과

### 4-1. 개인 사용자 관점

- 회의 중:
  - **“지금 무슨 말 나왔는지”에만 집중**할 수 있음
  - 동시에 실시간 로그 + 메모로 나만의 정리도 가능
- 회의 후:
  - 요약본 PDF, 메모 TXT, 녹음 파일이 모두 남아서
  - 복습 / 회고 / 회의록 공유가 편해짐
- 실무나 공부(스터디, 강의, 세미나)에서도
  - 강의 내용을 실시간으로 받아 적고
  - 끝나고 요약 PDF와 메모로 정리할 수 있는 도구로 활용 가능

### 4-2. 팀 / 조직 관점

- **회의 품질** 향상
  - “누가 어떤 얘기를 했는지”가 기록으로 남아  
    오해나 기억 오류를 줄일 수 있음
  - 각 주제별로 중간 정리가 남아서  
    회의가 산으로 가지 않고 **논의 흐름을 잡아줌**
- **회의 효율** 향상
  - 회의록 담당자를 따로 정하지 않아도 됨
  - 회의가 끝난 직후, 요약본과 메모를 바로 공유 가능
- **온보딩 / 히스토리 관리**
  - 신규 인원에게 과거 회의 요약 PDF를 공유해서  
    이슈 히스토리와 의사결정 과정을 빠르게 전달 가능
